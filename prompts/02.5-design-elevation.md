Read `00-orchestrator.md` rules first. Skills: `rtl-hebrew-webapp`, `catalog-and-filters`,

**Read DESIGN.md §6b and §7 IN FULL — they define this phase.** Every budget in §6b is a

Mobile is the primary target. Build and verify everything at 390px + DevTools mobile
throttling FIRST; desktop is the enhancement.

## Track A ∥ — Living cat cards (flagship) + cat video schema

1. Migration `000X_cat_video.sql`: add `video_path text` (nullable) to `cats` —
   one optional clip per cat, stored in the `cat-photos` bucket under
   `{cat_id}/{uuid}-clip.(webm|mp4)`. Regenerate DB types. Log in ARCHITECTURE §11.
2. Extend the seed: attach openly-licensed clips to 4–6 of the 12 published cats
   (Pexels/Mixkit video, transcode with the existing ffmpeg tooling to §6b's clip spec:
   ≤3s, 480px, muted, WebM+MP4, ≤1MB, first frame matching the cover crop). Record
   source URLs in the seed script comments.
3. `CatCard`: video-capable per §6b — poster (cover photo) by default; a single
   site-wide "playback director" (small client module) decides which ONE card plays:
   viewport-center on mobile, hover/focus on desktop. Cards without video are untouched.
4. Prove the budget: DevTools network log on the catalog at 390px — only one clip
   downloads at a time; scrolling pauses the previous card. Data Saver ⇒ zero video
   requests.

## Track B ∥ — Hero film sequence + ambient life (landing page)

1. Replace the single hero loop with the 3-clip crossfade sequence per §6b (only clip #1
   eager; #2–#3 lazy after `load`; 1.5s crossfade; captions optional). Reuse/extend
   `scripts/transcode-video.mjs`. Sources: warm, natural-light cat clips (Pexels/Mixkit),
   source URLs in comments.
2. FIX the outstanding reduced-motion violation: under `prefers-reduced-motion` the hero
   renders the poster only — no video request at all (also honor `saveData`).
3. Micro-life: logo ח-ears twitch (~20s interval, SVG transform, pauses under
   reduced-motion) · CSS sunbeam drift on paper sections (§6b spec).

## Track C ∥ — Morph transitions + self-drawing mascot

1. View Transitions: card→detail shared-element morph (the cover photo is the shared
   element), 250–300ms. Feature-detect; unsupported browsers get normal navigation.
   Reduced-motion ⇒ instant navigation.
2. Self-drawing mascot: stroke-dashoffset draw-on (~700ms) triggered by
   IntersectionObserver — apply in how-it-works (the cat "draws itself" as steps enter)
   and catalog empty state. Static final frame under reduced-motion.

## Global playback rules (whoever touches video)

One `<video>` playing site-wide at any moment · off-screen ⇒ pause · `document.hidden`
⇒ pause · `saveData` ⇒ posters only · every video: `muted playsInline loop
preload="none"` (except hero clip #1: `preload="metadata"`), explicit poster.
