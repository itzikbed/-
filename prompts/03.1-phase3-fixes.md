## Finding 1 — BUG (high): publisher-uploaded videos never play anywhere

The wizard stores `cats.video_path` WITH its file extension
(`{catId}/{uuid}-video.mp4` — components/cats/UploadStep4.tsx). Both playback
components append ANOTHER extension pair:

- components/cats/CatCard.tsx (~line 155): `src={...${cat.video_path}.webm}` / `.mp4`
- components/cats/GalleryVideoPlayer.tsx (~lines 87-88): same pattern

So publisher uploads request `...-video.mp4.webm` → 404 → the video silently
never plays, not in the catalog and not on the detail page. Only the seeded
Phase 2.5 clips (extension-less base path with a pre-optimized .webm/.mp4 pair)
work.

Required behavior (ARCHITECTURE §11 video policy v1):

- `video_path` WITH extension ⇒ RAW publisher upload ⇒ plays ONLY on the
  detail page (GalleryVideoPlayer), tap-to-play, a single `<source>` with the
  exact stored path (no extension appending).
- `video_path` WITHOUT extension ⇒ pre-optimized pair ⇒ current behavior
  (catalog autoplay via PlaybackDirector + detail page, both sources).
- CatCard must NOT render the video element or the video-affordance badge for
  raw (extension-bearing) paths — poster only.

Put the "has extension?" helper in one shared util, not duplicated.
Acceptance: upload a short mp4 through the wizard on a PRODUCTION build; it
plays on tap in the detail page; the catalog card shows the poster with no
video badge; seeded cards still autoplay on hover.

## Finding 2 — a11y: wizard radio groups are one giant keyboard group
