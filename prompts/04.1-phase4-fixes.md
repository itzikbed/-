## Finding 1 — SECURITY (HIGH): open redirect on login

`components/auth/LoginForm.tsx:19` reads `searchParams.get('redirect')` and
`:43` passes it straight to `router.push(redirectUrl)` with no validation.
`/login?redirect=https://evil.com/phish` (or protocol-relative `//evil.com`)
bounces a just-authenticated user off-site — classic credential-phishing open
redirect. Verified in code by the architect.

Fix: accept only same-origin relative paths. A path is safe iff it starts with
a single `/` and NOT `//` (and not `/\`). Anything else ⇒ fall back to `/`. Put
the guard in one helper (e.g. `lib/utils/safe-redirect.ts`) with a unit test
covering: `/admin` ✓, `/cats?x=1` ✓, `//evil.com` → `/`, `https://evil.com` →
`/`, `/\evil.com` → `/`, `javascript:alert(1)` → `/`, empty/null → `/`. Apply
it wherever the redirect param is consumed.

## Finding 2 — correctness (MEDIUM): awaited email lookup breaks fire-and-log + idempotency

In the admin actions, `getUserEmail()` (`app/admin/actions-helper.ts:19-24`)
THROWS on failure and is `await`ed BEFORE the fire-and-log `sendEmail` (e.g.
`app/admin/request-actions.ts:44-45`, `cat-actions.ts:38`,
`publisher-actions.ts:37`) — after the status UPDATE and moderation_log insert
have already committed. A failed address lookup then returns `{ok:false}`, and
on retry the status guard fails, so the SAME admin who actually succeeded is
shown "טופל כבר על ידי אדמין אחר".

Fix: address resolution must be part of the fire-and-log tail, never fatal to
the decision. Resolve the email inside (or alongside) the `sendEmail` path and
swallow+log failures; the action returns `ok:true` once the DB decision +
moderation_log succeed, regardless of email/address outcome. Never send an
email on a lost race (unchanged — keep it after the guard).

## Finding 3 — concurrency (MEDIUM): publisher optimistic guard too permissive

`app/admin/publisher-actions.ts:21` guards with
`.in('publisher_status', ['pending','none','blocked'])` and `:65` with
`['pending','none','approved']`. Two admins acting on the same publisher (one
approve, one reject) both match and both "succeed" — no conflict message, and
the final state can contradict an approval email already sent. Cats/requests
use strict `status='pending'` and are correct; make publisher match them: guard
strictly on the single expected starting state (`publisher_status='pending'`)
so a lost race returns the Hebrew conflict message.

## Finding 4 — RLS hardening (MEDIUM): RPC EXECUTE not revoked from PUBLIC/anon

`supabase/migrations/0005_contact_handoff_rpc.sql` grants EXECUTE to
`authenticated` but never revokes the default PUBLIC grant, so `anon` retains
EXECUTE (harmless today — anon's `auth.uid()` is null ⇒ zero rows — but a
compliance gap). Add, in a NEW migration `0006_*.sql` (never edit an applied
migration): `revoke execute on function public.get_handoff_contact(uuid) from
public, anon;` Also add `alter function public.get_handoff_contact(uuid) owner
to postgres;` so SECURITY DEFINER definitively bypasses `profiles` RLS
regardless of the role that ran the migration.

## Finding 5 — test integrity (MEDIUM): TEST 12 doesn't assert its invariant

`scripts/rls-smoke.mjs` TEST 12 is titled "invariant verification" and logs it,
but only deletes rows — it never re-queries `cats` to assert exactly 12
published after promoting `draftCat` to a 13th. Fix: after cleanup, actually
assert the published count === 12 and fail loudly otherwise. Also wrap the
mutate→verify→cleanup span (TESTS 8-12) so a mid-test `process.exit` cannot
leave a 13th published cat or orphaned users (try/finally that deletes the test
users + the promoted cat). Negative assertions at the isolation checks
(lines ~459/466/473/480/503/509/540) must also fail on an unexpected `error`,
not treat it as "empty".

## Finding 6 — email (MEDIUM): buttons are not Outlook-bulletproof

All six templates put padding on the `<a>` (`buttonLinkStyle`) while the `<td>`
carries only background+border-radius. Outlook's Word engine ignores padding on
inline `<a>` and ignores `border-radius`, collapsing the CTA to a bare square
link. Move padding onto the `<td>` (and add an MSO VML roundrect fallback, or at
minimum a mso-padding conditional) so the primary CTA renders as a real button
in Outlook. Israel has heavy Outlook/desktop usage — this is the money button.

## Finding 7 — dev-script safety (MEDIUM): bootstrap-admin hardcodes creds, no env guard, silent role race

`scripts/bootstrap-admin.mjs` mints `arch-review-admin@example.com` /
`Review!2026` against whatever `.env.local` points to, and sets `role='admin'`
via `.update().eq()` after a fixed `setTimeout(1000)` with NO affected-row
check — on a slow signup trigger it prints success while the role was never set.
Fix: (a) refuse to run unless `NEXT_PUBLIC_SUPABASE_URL` is a localhost/127.*
URL (hard exit otherwise); (b) after the role update, assert exactly one row
changed (re-select and check `role==='admin'`), retry/loop instead of a blind
sleep; (c) keep it out of any build/prod path.
